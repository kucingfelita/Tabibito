# PROMPT PROJECT: SISTEM RESERVASI TIKET WISATA & MARKETPLACE

## 1. DESKRIPSI PROYEK & TECH STACK
Saya ingin membuat website Marketplace Pemesanan Tiket Wisata yang komprehensif.
- **Backend:** Laravel 11/12 (PHP)
- **Frontend:** Laravel Blade, Tailwind CSS, Alpine.js (untuk interaktivitas ringan seperti dropdown/modal/AJAX).
- **Database:** MySQL
- **Payment Gateway:** Midtrans (Snap API & Webhook/Notification).
- **Fitur Tambahan:** Socialite (Google Login), Laravel Mail (via SMTP) yang dijalankan menggunakan Laravel Queues (Background Jobs), dan pembuatan QR Code (menggunakan library simple-qrcode).

## 2. STRUKTUR DATABASE (MIGRATIONS)
Buat struktur database dengan tabel utama berikut:
1. `users`: id, name, username (unique), email (unique), password, phone (nullable), google_id, tipe_user (1=Admin, 2=User, 3=Owner), balance (default 0).
2. `destinations`: id, user_id (foreign key ke tipe_user 3), name, description (text panjang), address, city, map_link, open_time, close_time, status (pending/active/rejected).
3. `tags`: id, name.
4. `destination_tag`: id, destination_id, tag_id.
5. `destination_images`: id, destination_id, image_path (untuk galeri foto).
6. `tickets`: id, destination_id, name, price, benefit (text), daily_quota, current_quota.
7. `transactions`: id, order_id (unique string untuk midtrans), user_id, ticket_id, qty, total_price, booking_date, status (pending/settlement/expire/used), snap_token, qr_code_token (unique).
8. `withdrawals`: id, user_id, amount, admin_fee, ewallet_or_bank_name, account_number, status (pending/approved/rejected).

## 3. GLOBAL USER INTERFACE (UX/UI)
- **Desain:** Modern, bersih, menggunakan komponen Tailwind (seperti Cards, Badges untuk Tag).
- **Navbar:** 
  - Logo di kiri. 
  - Jika Guest: Tampilkan tombol "Sign Up" & "Login".
  - Jika Auth: Tombol Sign Up/Login hilang, diganti Avatar/Icon Profile. Jika di-klik memunculkan dropdown (Profile, Riwayat, Dashboard (jika owner/admin), Logout).
- **Footer:** Links ke sosial media, menu navigasi, halaman kontak, copyright.
- **Breadcrumbs:** Tambahkan navigasi breadcrumbs di halaman dalam (Contoh: Beranda > Wisata > Candi Borobudur).
- **Notifikasi/Alert:** Gunakan SweetAlert2 atau Toast notification untuk pesan sukses/gagal.
- **Loading State:** Gunakan "Skeleton Loading" (kotak abu-abu animasi berdenyut) saat memuat data wisata/tiket menggunakan AJAX.

## 4. ALUR AUTENTIKASI (AUTH)
- **Login:** Form berisi Input (Username atau Email) dan Password. Tombol utama "Login", dan tombol alternatif "Login with Google".
- **Sign Up:** Form berisi Nama, Username, No HP, Password, Konfirmasi Password. Di bawah form ada teks: "Sudah punya akun? Login disini".
- **Google Auth Handling:** Jika user daftar via Google, otomatis buatkan akun dengan `tipe_user = 2`. Jika belum ada password/No HP, beri penanda di database agar user diminta melengkapinya nanti di halaman Profile.
- **Email Event:** Kirim **Welcome Email** (HTML mailable cantik) saat user berhasil mendaftar.

## 5. HALAMAN PENGGUNA (FRONTEND)
- **Beranda:**
  - Hero banner (gambar besar, judul, kotak pencarian lokasi).
  - Seksi "Rekomendasi": Tampilkan 3 card wisata terbaik dengan label "*Mulai dari Rp XX.XXX*".
  - Seksi "Cara Pesan": 3 step tata cara pesanan (Pilih -> Bayar -> Scan QR).
- **Halaman Semua Wisata:**
  - Sidebar/Top bar untuk **Filter**: Berdasarkan Tag (Pantai, Gunung, dll) dan Daerah (Kota).
  - Tampilan List: Card berbentuk Persegi Panjang (Horizontal). Berisi Foto Utama (kiri), Nama, Tag (berupa pil/badge), Deskripsi Singkat (terpotong), Daerah (kanan).
  - Pagination: Hanya tampilkan 20. Jika lebih, berikan tombol "Lebih Banyak". Gunakan AJAX (tanpa refresh halaman).
- **Halaman Detail Wisata:**
  - Bagian atas: Galeri Foto, Judul, Lokasi, Jam Buka, Tombol "Arahkan via Google Maps".
  - Bagian tengah: Deskripsi lengkap dan Tag.
  - Bagian bawah (Tiket): List card tiket (Load more per 10 tiket). Card berisi Nama Paket, Benefit, Sisa Kuota, Harga, dan tombol "Pesan Tiket".
- **Halaman Kontak:** Form hubungi admin, dan **Tombol Besar "Daftarkan Wisata Anda"** untuk user yang ingin menjadi Pemilik Wisata.

## 6. ALUR CHECKOUT & PEMBAYARAN (CRITICAL)
- **Validasi:** Saat user klik "Pesan Tiket", jika belum login lempar ke Halaman Login.
- **Form Checkout:** 
  - Input: Tanggal Booking (wajib, tidak bisa pilih tanggal masa lalu) dan Jumlah Tiket (maksimal sesuai `current_quota` saat ini).
  - Info: Nama pemesan (otomatis terisi/readonly), Rincian pesanan (Harga x Jumlah).
  - Tombol: "Bayar Sekarang".
- **Escrow & Midtrans Logic:**
  - Saat klik bayar: Validasi ulang kuota di *controller*. Jika aman, generate `order_id` dan minta `snap_token` ke Midtrans.
  - Tampilkan pop-up Midtrans (Snap.js).
  - **Webhook (Notification Handler):** Midtrans akan hit endpoint web kita secara otomatis di background.
    - Jika `status == settlement`: Ubah status transaksi jadi 'settlement', kirim **E-Ticket Email** (berisi PDF tiket & QR Code), dan **tambahkan saldo (total_price) ke balance milik Owner (user_id pembuat destinasi)**.
    - Jika `status == expire/cancel`: Kembalikan `current_quota` tiket seperti semula.
- **Email Event (Queued):** Kirim Email Invoice saat Pending, dan Email Tiket Sukses saat Settlement.

## 7. DASHBOARD USER BIASA (TRAVELER)
- **Profile:** Edit biodata, tambah No HP, Ganti Password.
- **Riwayat Pembelian:** List tiket. Jika di-klik, muncul detail. Jika tiket statusnya Lunas dan belum digunakan, tampilkan **QR Code** besar yang siap di-scan petugas.

## 8. ALUR PEMILIK WISATA (OWNER / SELLER)
- **Pendaftaran (Onboarding):** Dari halaman kontak, user mengisi form: Nama Wisata, Tag, Domisili. Sistem mengubah status wisata jadi 'Pending'.
- **Dashboard Analytics:** Grafik total tiket terjual, sisa kuota hari ini, dan total pendapatan.
- **Manajemen Wisata:** Edit deskripsi, jam buka, tambah foto galeri. Gunakan package *Image Intervention* untuk compress gambar upload max 1MB.
- **Manajemen Tiket:** CRUD tiket. *Logika Khusus:* Buat Laravel Scheduler (Cron Job) yang berjalan setiap jam 00:00 untuk mereset nilai `current_quota` kembali sama dengan `daily_quota`.
- **Scanner Tiket (Fitur Utama):** Halaman khusus berbasis web yang menyalakan kamera device untuk **Scan QR Code**. Saat di-scan, cek `qr_code_token` di database, jika valid ubah status transaksi dari 'settlement' menjadi 'used' (Tiket Hangus/Telah Terpakai).
- **Keuangan (Withdraw):** 
  - Halaman berisi sisa saldo berjalan.
  - Form Tarik Dana: Pilih Bank/E-Wallet, Nomor Rekening.
  - Logika: Minimal tarik Rp 50.000. Biaya admin 5% (Total tarik = Jumlah - 5%).
  - Memotong saldo otomatis dan status withdraw menjadi 'Pending' (menunggu transfer Admin).

## 9. DASHBOARD ADMIN WEBSITE
- **Verifikasi Destinasi:** List pendaftaran wisata baru. Tombol "Approve" (otomatis ubah tipe_user pendaftar jadi 3/Owner) atau "Reject".
- **Manajemen User:** List semua user, bisa suspend akun.
- **Manajemen Pencairan Dana:** List request Withdraw. Tombol "Tandai Selesai" (setelah admin mentransfer uang manual/lewat API Payout ke rekening owner).
- **Email Event:** Kirim notifikasi email ke Owner saat tempat wisata disetujui, dan saat pencairan dana berhasil ditransfer.

PENTING: Tuliskan kode dengan prinsip Clean Code, modularisasi controller, dan pastikan route diproteksi menggunakan Middleware sesuai `tipe_user`.
